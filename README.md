# Fraud Detection System

A production-grade fraud detection system built with LightGBM + Neural Network ensemble, explainability via SHAP, and a business cost framework at its core.

## What this project demonstrates

- **Why a 1% false-positive rate is catastrophic at scale** — the dashboard shows this in real numbers
- Time-based train/val/test splitting to simulate concept drift (no data leakage)
- Feature engineering: velocity features, behavioural deviation, temporal patterns
- Stacked ensemble: LightGBM + MLP → Logistic Regression meta-learner
- SHAP explainability on individual predictions
- Threshold tuning driven by a business cost function, not raw accuracy
- A full model card documenting limitations, failure modes, and ethical considerations

## Stack

| Layer | Technology |
|---|---|
| Model training | Kaggle Notebooks (P100 GPU) |
| Feature engineering | pandas, scikit-learn |
| Models | LightGBM, PyTorch MLP, scikit-learn |
| Explainability | SHAP |
| API | FastAPI + uvicorn |
| Deployment | Railway (backend), Vercel (frontend) |
| Frontend | Next.js + Tailwind CSS + Recharts |

## Dataset

[IEEE-CIS Fraud Detection](https://www.kaggle.com/c/ieee-fraud-detection) — 590k transactions, 400+ features, 3.5% fraud rate.


## Reproducing the model

1. Download the IEEE-CIS dataset from Kaggle and place CSVs in `data/raw/`
2. Run notebooks 01 → 05 in order
3. Trained model artifacts land in `models/`
4. `cd api && uvicorn main:app --reload` to run the API locally

## Running locally

```bash
# Backend
cd api
pip install -r requirements.txt
uvicorn main:app --reload

# Frontend
cd frontend
npm install
npm run dev
```
